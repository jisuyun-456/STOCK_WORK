# Phase 2.5: Research Overlay — Wall Street Hybrid Trading System

> Date: 2026-04-09
> Status: Design Approved
> Predecessor: Phase 0 (fdd32fa) → Phase 1 (35e3db5) → Phase 2 (12ae3eb) → Alpaca (678af84)

## Context

Paper Trading 시스템의 Phase 0~2가 완료되었다. 현재 파이프라인은 Python 전략 모듈이 시그널을 생성하고 리스크 게이트를 거쳐 Alpaca에 주문하는 구조다.

**문제:** Python 모듈의 정량 시그널만으로는 펀더멘탈, 매크로, 기술적 분석 등 정성적 판단이 빠진다. 단순히 모멘텀 수치가 높다고 바로 매수하기보다, 다각도 리서치를 거쳐 확신도를 보정해야 한다.

**해결:** Research Division 5명 에이전트를 추가하여 Python 시그널에 대해 병렬 리서치 → 합의 기반 confidence 보정 → 리스크 FAIL 시 재심(appeal) 피드백 루프를 구축한다.

**결과:** Trading Desk 5명 + Research Division 5명 = 10 에이전트 하이브리드 시스템.

## 1. Pipeline Architecture

기존 7-phase에 Phase 2.5 RESEARCH와 Phase 3.5 APPEAL을 삽입한다.

```
Phase 1:   DATA        → yfinance + Alpaca positions → snapshot
Phase 2:   SIGNALS     → Python 전략 모듈 → raw Signal[]
Phase 2.5: RESEARCH    → Research Division 5명 병렬 → confidence 보정 (NEW)
Phase 3:   RISK        → 5가지 리스크 게이트
Phase 3.5: APPEAL      → Risk FAIL → Research 재심 → 최종 판정 (NEW)
Phase 4:   RESOLVE     → 충돌 해소
Phase 5:   EXECUTE     → Alpaca Paper API
Phase 6:   REPORT      → performance.json + daily report
Phase 7:   COMMIT      → git
```

### Phase 2.5 RESEARCH 흐름

```
raw signals (from Phase 2)
    │
    ├─ --research-mode=selective → confidence 0.5~0.7만 필터
    ├─ --research-mode=full     → 전체 (기본값)
    │
    ▼
5 Research Agents 병렬 실행
    │  Equity Research    → 밸류에이션 검증
    │  Technical Strategist → 차트/기술적 검증
    │  Macro Economist    → 매크로 체제 검증
    │  Portfolio Architect → 배분 최적화 검증
    │  Risk Controller    → 리스크 심층 검증 + VETO 권한
    │
    ▼
Weighted Consensus → adjusted confidence
    │
    ├─ VETO 발동 → 즉시 REJECT (appeal 불가)
    └─ 정상 → Phase 3로 전달
```

### Phase 3.5 APPEAL 흐름 (피드백 루프)

```
Phase 3 Risk FAIL signals
    │
    ▼
Research Division 재심 요청 (mode="appeal")
    │
    ├─ "리스크는 높지만 펀더멘탈이 압도적으로 강한가?"
    ├─ 5명 중 4명+ STRONG_OVERRIDE 시 → Risk Gate 1개 완화 허용
    │    완화 가능: sector_concentration(40%), portfolio_var(3%), correlation(0.85)
    ├─ override 절대 불가: position_limit(20%), cash_buffer(5%)
    └─ 재심은 1회만 (무한루프 방지)
    │
    ├─ OVERRIDE 승인 → Phase 4 RESOLVE로
    └─ OVERRIDE 거부 → 최종 REJECT + Dissent Log 기록
```

### 안전장치

- Risk Controller VETO = 절대적. Phase 2.5에서 VETO 발동 시 appeal 자체 불가
- position_limit(20%)과 cash_buffer(5%)는 appeal override 불가 (하드코딩)
- Appeal 1회 제한 — 재심의 재심 없음
- 모든 appeal 결과는 `state/dissent_log.jsonl`에 Immutable 기록

## 2. Research Division — 에이전트 5명

### 2.1 Equity Research (equity-research.md)

**원본:** archive/stock-reports-v1 `fundamental-analyst.md` 파트 A
**모델:** claude-sonnet-4-6

핵심 도메인 지식:
- DCF (FCFF/FCFE), DDM (Gordon/H-Model/3-stage), Relative Valuation
- 민감도 분석: WACC ±1%, g ±0.5% 매트릭스 (5x5)
- Beneish M-Score, Altman Z-Score, Piotroski F-Score
- DuPont 5-Factor, Quality of Earnings (OCF/NI > 1.0)
- SEC (10-K/10-Q/8-K/13F), DART 공시 해석

Paper Trading 특화:
- ResearchRequest 수신 → 해당 종목 밸류에이션 검증
- Margin of Safety 계산 (현재가 대비 적정가)
- ResearchVerdict 출력 (AGREE/DISAGREE + confidence_delta)

### 2.2 Technical Strategist (technical-strategist.md)

**원본:** archive/stock-reports-v1 `market-scanner.md` 파트 B + `quant-strategist.md` 기술 분석 부분
**모델:** claude-sonnet-4-6

핵심 도메인 지식:
- CMT III 수준: Wyckoff, Elliott Wave, VSA, Market Profile
- 변동성: IV Surface, VIX Contango/Backwardation, VKOSPI
- Greeks: Delta/Gamma/Theta/Vega/Rho
- 레버리지 수학: Volatility Drag = L² × σ² / 2
- 괴리율 (NAV vs 시장가): ±1% 정상, ±3% 경고

Paper Trading 특화:
- 종목별 기술적 지지/저항/추세 검증
- 진입 타이밍 적절성 판단
- LEV 전략 시그널에 대해 변동성 끌림 경고

### 2.3 Macro Economist (macro-economist.md)

**원본:** archive/stock-reports-v1 `fundamental-analyst.md` 파트 B
**모델:** claude-sonnet-4-6

핵심 도메인 지식:
- Taylor Rule, Fed Dot Plot, BOK 금통위
- NBER 4단계 경기사이클, ISM PMI, LEI
- Dalio 장기부채사이클, Minsky Moment
- CPI/PCE, BEI, DXY, 원/달러
- BOK ECOS, KDI 경제전망

Paper Trading 특화:
- **Regime Detection** (Phase 2.5 시작 전 자동 실행):
  - S&P500 < SMA200 & VIX > 30 → CRISIS
  - S&P500 < SMA200 & VIX ≤ 30 → BEAR
  - S&P500 > SMA200 & VIX < 20 → BULL
  - 그 외 → NEUTRAL
- 매크로 환경이 시그널 방향과 일치하는지 검증
- 금리/환율 변동이 특정 섹터에 미치는 영향 분석

### 2.4 Portfolio Architect (portfolio-architect.md)

**원본:** archive/stock-reports-v1 `chief-strategist.md` 배분 관련 + 신규
**모델:** claude-sonnet-4-6

핵심 도메인 지식:
- MPT (Modern Portfolio Theory), Efficient Frontier
- Black-Litterman Model (시장 균형 + 주관적 뷰)
- HRP (Hierarchical Risk Parity)
- Kelly Criterion (최적 베팅 비율)
- Core-Satellite 전략

Paper Trading 특화:
- 4전략(MOM/VAL/QNT/LEV) 간 자본 배분 검증
- 새 시그널이 전체 포트폴리오 효율성에 미치는 영향
- 상관관계 클러스터 분석
- 리밸런싱 필요성 판단

### 2.5 Risk Controller (risk-controller.md)

**원본:** archive/stock-reports-v1 `chief-strategist.md` Pre-Mortem + 신규
**모델:** claude-sonnet-4-6

핵심 도메인 지식:
- FRM (Financial Risk Manager) 수준
- Basel III/IV 자본적정성
- CVaR (Conditional Value at Risk), GARCH 변동성 모델
- Monte Carlo Simulation
- 2008 GFC / 2020 COVID 스트레스 시나리오
- Kahneman Pre-Mortem: "1년 후 이 판단이 틀렸다면 가장 큰 이유는?"

Paper Trading 특화:
- **VETO 권한**: 심각한 리스크 발견 시 즉시 REJECT (appeal 불가)
- VETO 기준:
  1. 파산 위험 (Z-Score < 1.81)
  2. 이익 조작 의심 (M-Score > -1.78)
  3. 극단적 변동성 (1일 VaR > 5%)
  4. 유동성 위험 (일 거래량 < 시그널 규모의 10%)
- Appeal 시 최종 판단자 역할

## 3. Weighted Consensus Algorithm

### 기본 가중치

```python
BASE_WEIGHTS = {
    "equity_research":      0.25,
    "technical_strategist": 0.20,
    "macro_economist":      0.20,
    "portfolio_architect":  0.15,
    "risk_controller":      0.20,
}
```

### Regime-Aware 동적 조정

```python
REGIME_MULTIPLIERS = {
    "CRISIS":  {"risk_controller": 2.0, "macro_economist": 1.5},
    "BEAR":    {"risk_controller": 1.5, "macro_economist": 1.3},
    "BULL":    {"equity_research": 1.3, "technical_strategist": 1.2},
    "NEUTRAL": {},  # 기본 가중치 유지
}
```

가중치 적용 후 정규화하여 합 = 1.0

### Confidence 보정 공식

```python
adjusted_confidence = original_confidence + Σ(normalized_weight_i × confidence_delta_i)
adjusted_confidence = clamp(adjusted_confidence, 0.0, 1.0)
```

특수 규칙:
- VETO 발동 → confidence = 0.0 (즉시 REJECT)
- DISAGREE 3명+ → 추가 -0.15 패널티
- AGREE 전원 (5/5) → 추가 +0.15 보너스
- 최종 confidence < 0.4 → 자동 DROP (Phase 3 진입 불가)

## 4. Data Structures

### ResearchRequest

```python
@dataclass
class ResearchRequest:
    signals: list[Signal]           # Phase 2 결과
    market_snapshot: dict           # Phase 1 데이터 (prices, positions)
    portfolio_state: dict           # state/portfolios.json
    regime: str                     # "BULL" | "BEAR" | "CRISIS" | "NEUTRAL"
    mode: str                       # "initial" | "appeal"
    appeal_context: dict | None     # appeal 시: {"failed_checks": [...], "risk_details": {...}}
```

### ResearchVerdict

```python
@dataclass
class ResearchVerdict:
    agent: str                      # "equity_research" 등
    symbol: str                     # 분석 대상 종목
    direction: str                  # "AGREE" | "DISAGREE" | "VETO"
    confidence_delta: float         # -0.3 ~ +0.3 보정값
    conviction: str                 # "STRONG" | "MODERATE" | "WEAK"
    reasoning: str                  # 1-3문장 근거
    key_metrics: dict               # 에이전트별 핵심 지표
    override_vote: str | None       # appeal 시: "STRONG_OVERRIDE" | "REJECT" | None
    timestamp: str                  # ISO 8601
```

### Regime Detection Output

```python
@dataclass
class RegimeDetection:
    regime: str                     # "BULL" | "BEAR" | "CRISIS" | "NEUTRAL"
    sp500_vs_sma200: float          # 비율 (1.05 = SMA200 대비 5% 위)
    vix_level: float                # 현재 VIX
    reasoning: str                  # 판정 근거
    timestamp: str
```

## 5. Research Cache

파일: `state/research_cache.json`

```json
{
  "NVDA": {
    "cached_at": "2026-04-09T22:30:00Z",
    "expires_at": "2026-04-16T22:30:00Z",
    "regime_at_cache": "BULL",
    "verdicts": [
      {"agent": "equity_research", "direction": "AGREE", "confidence_delta": 0.1, ...},
      ...
    ]
  }
}
```

규칙:
- TTL: 7일
- Regime 변경 시 전체 캐시 무효화
- `--no-cache` 플래그로 강제 새로고침
- 캐시 히트 시 에이전트 호출 스킵 → 토큰 절약

## 6. Dissent Log

파일: `state/dissent_log.jsonl` (append-only, Immutable Ledger)

```json
{
  "ts": "2026-04-09T22:35:00Z",
  "symbol": "INTC",
  "phase": "research",
  "majority_direction": "AGREE",
  "majority_count": 4,
  "dissenters": [
    {"agent": "risk_controller", "direction": "DISAGREE", "reason": "VaR 3.2% 초과 우려"}
  ],
  "final_outcome": "APPROVED",
  "appeal": false,
  "original_confidence": 0.72,
  "adjusted_confidence": 0.65
}
```

Performance Accountant가 월간 리포트에 포함하는 지표:
- Dissent 적중률: 소수 의견이 맞았던 비율
- VETO 정확도: VETO 발동 후 해당 종목의 실제 수익률
- Appeal 성공률: override 승인 후 수익/손실

## 7. Cost Guard

| 모드 | Research 호출 | 예상 토큰/사이클 |
|------|-------------|--------------|
| `--skip-research` | 0회 | ~5K |
| `--research-mode selective` | confidence 0.5~0.7만 | ~30K |
| `--research-mode full` (기본) | 전체 시그널 | ~80K |
| full + appeal 발생 시 | 전체 + FAIL 재심 | ~120K |

CLI 플래그:
- `--research-mode {full|selective|skip}` (기본: full)
- `--no-cache` (캐시 무시)
- dry-run 시 기본값 = selective
